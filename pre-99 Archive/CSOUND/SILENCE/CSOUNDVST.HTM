Introduction

CsoundVST is a Virtual Studio Technology (VST) plugin version of Csound, the well-known academic sound processing language. For an introduction to Csound, visit its home page. For an introduction to VST, visit the Steinberg web site.

CsoundVST is currently based on Csound version 4.17 from Bath. However, CsoundVST has several features not found in other versions of Csound:

      It works either as a synthesizer or as an effect in VST hosts such as Cubase or Orion Pro.

      It has a standalone Windows version with a graphical user interface for managing and editing Csound files.

      It has a C API for use in C programs.

      It has a C++ API for use in C++ programs. This API includes score management.

      It has a Java API (Java Native Interface or JNI) for use in Java programs; this is what Silence uses.

      It has a MathLink interface for use with Mathematica 4.1 for mathematics-based algorithmic composition.

      It has its own plugin format for Csound opcodes.

Installation

CsoundVST is tested with Windows 2000, and probably works with Windows XP, Windows NT, Windows ME, Windows 98 Second Edition, and Windows 95.

Create a directory called Silence, and unzip csoundvst.zip into it, preserving pathnames.

Create a SILENCE_HOME environment variable that points to the Silence directory.

Create a CSOUND_HOME environment variable that points to %SILENCE_HOME%/bin (CsoundVST uses it to load plugin opcodes).

Create a STK_HOME environment variable that points to %SILENCE_HOME% (CsoundVST’s plugin opcodes use it to load waveform and sample files).

If you want to use CsoundVST as a VST plugin, place a copy of CsoundVST.dll into your VST shared plugins directory. To verify your installation, if you have Cubase 32, you can load Silence/examples/CsoundVST/SynthesizerSample.all and just start it playing. Otherwise, create or import a MIDI track into your VST host, create an instance of CsoundVST, assign it to the MIDI track, and open the Silence/examples/CsoundVST/MidiTest.csd file. Start the host playing, and you should hear a piano-type sound playing your track. If you hear nothing, click on the Perform VST button and try again.

If you want to run CsoundWindows as a standalone Windows version of Csound, make sure that %SILENCE_HOME%/bin is in your Windows PATH. To verify your installation, run CsoundWindows, open the Silence/examples/CsoundVST/trapped.csd file, click on the Perform Csound button, and when performance is finished, play the soundfile.

Note that if you want to rebuild CsoundVST or to call it from your own software, you must obtain the VST2 software development kit from Steinberg, which is available here. Copy all of the files in the vstsdk2/source/common directory into your %SILENCE_HOME% /src/cpp/CsoundVST directory.

Operation

CsoundVST can be used in several different modes:

      As a VST plugin.

      As a standalone Windows version of Csound.

      As a software synthesis library for building your own music software.

In all cases, CsoundVST has the same graphical user interface, shown below.

The functions of the buttons are as follows:

  1. New – Removes all current data from CsoundVST.
  2. Open – Loads a csd, orc, sco, or MIDI file, replacing all current data.
  3. Import – Loads an orc, sco, or MIDI file, replacing only the respective orc, sco, or MIDI data.
  4. Save – Saves the current data.
  5. Save as – Saves the current csd, orc, sco, or MIDI data into a file that you specify.
  6. Edit command – Displays the Csound command in the text box for editing.
  7. Edit orchestra – Displays the complete Csound orchestra in the text box for editing.
  8. Edit score – Displays the complete Csound score in the text box for editing.
  9. Edit arrangement – Displays a page for editing Csound arrangements. This facility is unique to CsoundVST. It assumes that each instrument in the orchestra has a command immediately following the instrument number with the name of the instrument, for example instr 1 ; Flute. The instrument names are parsed out and put into a list to pick from and rearrange. If such an arrangement is defined, Csound exports the named instruments in the order chosen for performance, instead of the entire orchestra file.
  10. Output – Displays a text box that records Csound’s performance messages.
  11. Perform Csound – Exports the orc, sco, and MIDI data, then performs them just as standard Csound would do.
  12. Perform VST – Exports the orc, sco, and MIDI data, then performs them for the VST host.
  13. Stop performing – Cancels the current performance and resets Csound for the next performance.
  14. Play soundfile – Attempts to play the output soundfile using the Windows shell command “play”.
  15. Edit soundfile – Attempts to edit the output soundfile using the Windows shell command “edit”.

VST Synthesizer

Strictly speaking, CsoundVST is an “effect,” not a “synthesizer.” However, it is usually used as a synthesizer.

Set up CsoundVST as an effect in your VST host. Most hosts provide a means for using an effect as a synthesizer, or of sending MIDI events from a given MIDI channel to the effect.

Write a MIDI instrument definition for each channel that you will be receiving, and write audio out through the outs opcode. CsoundVST will receive MIDI events from the VST host as though it were a standard MIDI input port, and the host will receive all audio written to outs.

Set the Csound command to CsoundVST -h -n –d temp.orc temp.sco, so that there is no header on the audio output stream, and sound is written to the VST host and not to a soundfile. Once you get your orchestra working, you should use the –m0 option to turn the message output off.

If you want your Csound orc, sco, and other data to be saved in your VST host’s file, you must save the patch as an Instrument using the host’s facility.

VST Effect

This facility currently does not appear to work reliably.

Set the Csound command to CsoundVST -h -n –d temp.orc temp.sco, so that there is no header on the audio output stream, and sound is written to the VST host and not to a soundfile. Once you get your orchestra working, you should use the –m0 option to turn the message output off for greater efficiency.

To use CsoundVST as an effect, create an instrument definition in your Csound orchestra. Read audio in through the ins opcode, process it in your instrument, and write audio out through the outs opcode. Turn this instrument on at the beginning of your score, and leave it on for the duration of your performance.

If you want your Csound orc, sco, and other data to be saved in your VST host’s file, you must save the patch as an Instrument using the host’s facility.

Synthesis Engine Shared Library

Because CsoundVST is a Windows dynamic link library (DLL), or shared library, it can be loaded and called from other software, using a C API, a C++ API, or the VST plugin.

The CsoundConsole program shows how to use the C API.

The SSoundConsole program shows how to use the VST plugin and the C++ API.

The CsoundWindows program shows how to use the VST plugin and the C++ API, and how to place the CsoundVST GUI on an existing window.

Plugin Opcodes

See the header file for an explanation of this feature. Users can subclass the CsoundOpcodeBase class to write Csound opcodes in C++, and they will automatically be loaded by CsoundVST at run time. Two sample opcode plugins are included in the distribution, and are documented below.

csoundFreeverb.OPC

freeverb

ar, ar freeverb iroomsize, idamp, iwet, idry, iwidth, imode

Description

A port of Jezar’s carefully tuned Schroeder/Moorer stereophonic reverberation plugin to Csound using the original C++ code, without modification. There are 8 comb filters and 4 allpass filters for each channel.

Initialization

iroomsize – room depth, normalized from 0 to 1.

idamp – damping, normalized from 0 to 1.

iwet – wet signal gain, normalized from 0 to 1.

idry – dry signal gain, normalized from 0 to 1.

iwidth – room width, normalized from 0 to 1.

imode – reverberation mode, 0 for normal or 1 for freeze.

United States

Description

Perry Cook’s Synthesis Tool Kit (STK) physical modeling C++ instruments ported verbatim to Csound. To my ears, these sound a little better than the existing Csound ports.

Initialization

ifrequency – frequency of the note in Hertz, typically obtained from cpsmidi.

iamplitude – normalized amplitude of the note, typically obtained from veloc 0, 1.

Initialization values can also be derived from a score file or generated in the instrument.

Performance

kstatus – MIDI channel message status nybble, typically obtained from midiin.

kchannel – MIDI channel message channel nybble, typically obtained from midiin.

kdata1 – first MIDI channel message data byte, typically obtained from midiin.

kdata2 – second MIDI channel message data byte, typically obtained from midiin.

Control values can also be derived from a score file or generated in the instrument.

When using MIDI input, you will probably also want to use the linenr opcode to generate a damping envelope in order to extend performance beyond p3 with a decay tail to prevent clicking on release.

Changes in This Version

The CsoundVST source code is now synchronized with Csound version 4.17 from John ffitch.

CsoundVST has been reworked to use Matt Ingalls’ re-entrant playevents code. As a result, it is more efficient and more stable.

CsoundVST has been changed from a “synthesizer” to an “effect”, so that it can actually be used either as a synthesizer or as an effect. However, at this time, CsoundVST does not actually seem to work reliably as an effect.

The CsoundVST GUI has been rewritten to use native Win32 API calls, and an arrangement page has been added.

Plugin opcodes for Csound have been implemented, with two samples: Jezar’s freeverb, and Perry Cook’s complete Synthesis Tool Kit (STK).

A native MathLink interface has been created, in CsoundMathLink.exe, to simplify using CsoundVST from Mathematica 4.1, which is a powerful tool for algorithmic composition.

A number of minor bugs have been fixed.
