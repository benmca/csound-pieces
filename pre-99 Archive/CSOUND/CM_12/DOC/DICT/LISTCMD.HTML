list {objects}+ [Command]

Prints {objects} to the terminal. If {objects} is not specified, the current focus container is listed. Otherwise, for each reference in {objects}, if it is to a container then its subobjects are listed, otherwise each reference can be a range, iteration or grouping or container to list. Since listing a container without specifying a range could potentially list thousands of objects, the global parameter *listing-length* controls how many subobjects should be listed if no length is specified. The default value of *listing-length* is 50 objects.

Each listed object appears together with its position in the container from which it is listed. The list command uses the standard lisp function PRINT-OBJECT to print the objects to the terminal. When defining a new object class, it is a good idea add a method to PRINT-OBJECT that shows useful information when objects of this type are printed. Important features about these objects can then be viewed simply by using list.
