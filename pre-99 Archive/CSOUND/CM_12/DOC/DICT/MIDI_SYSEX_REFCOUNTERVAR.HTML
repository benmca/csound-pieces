*midi-sysex-refcounter* [Variable]

Global system exclusive message data reference counter. If a user schedules a system exclusive message for output at a later time, but wishes to dispose of the data as soon as all messages pointing to the block of data has been written out completely, she must either pass a private counter object created by calling

mi_d:make-refcounter initial-value
and monitor its value (or the value of the global *midi-sysex-refcounter* by calling
mi_d:refcounter-value refcounter
See midi-write-sysex for a discussion of how to use refcounter objects.
